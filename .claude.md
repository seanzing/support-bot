# Claude Code Documentation - Zing Customer Support Agent

This document provides context for Claude Code when working on this project.

## Project Overview

This is a production AI-powered customer support agent for **Zing Business Management Software**. It uses OpenAI's ChatKit and Agents API to provide intelligent support through knowledge base search, automated ticket creation to HubSpot, and real-time session tracking.

**IMPORTANT:** This project was converted from an airline demo (OpenSkies) to a Zing support agent. All airline references have been removed.

## Architecture

### Backend (FastAPI + OpenAI Agents)
- **Main Entry:** `backend/app/main.py` - FastAPI server with ChatKit endpoint
- **AI Agent:** `backend/app/zing_support_agent.py` - OpenAI Agent with 2 tools (search_knowledge_base, create_support_ticket)
- **Knowledge Base:** `backend/app/knowledge_base.py` - 40+ FAQs across 9 categories
- **Ticket Escalation:** `backend/app/hubspot_integration.py` - Email-based ticket creation via SMTP2GO
- **AI Insights:** `backend/app/ai_insights.py` - Sentiment analysis and issue classification
- **Email Templates:** `backend/app/email_templates.py` - Professional HTML ticket emails
- **State Management:** `backend/app/zing_state.py` - Session tracking and statistics
- **Memory Store:** `backend/app/memory_store.py` - Conversation history storage

### Frontend (React + TypeScript + Vite)
- **Main Layout:** `frontend/src/components/Home.tsx` - Responsive grid layout
- **Chat Interface:** `frontend/src/components/ChatKitPanel.tsx` - OpenAI ChatKit wrapper
- **Context Panel:** `frontend/src/components/CustomerContextPanel.tsx` - Session stats and activity
- **Theme Toggle:** `frontend/src/components/ThemeToggle.tsx` - Dark/light mode
- **Custom Hook:** `frontend/src/hooks/useCustomerContext.ts` - Polls `/support/customer` endpoint
- **Styling:** `frontend/src/index.css` - Tailwind + custom animations, glass morphism, shadows

## Tech Stack

**Backend:**
- Python 3.11+ with FastAPI
- OpenAI Agents SDK (configurable model via ZING_SUPPORT_MODEL env var)
- SMTP2GO for email-based ticket escalation (triggers HubSpot workflow)
- Server-sent events (SSE) for streaming

**Frontend:**
- React 18 + TypeScript
- Vite for builds
- Tailwind CSS + shadcn/ui components
- OpenAI ChatKit Web Component
- Custom animations and glass morphism effects

## Key Features

### AI Agent Tools (2 total)
1. **search_knowledge_base** - Searches 40+ FAQs with semantic matching, logs to zing_state
2. **create_support_ticket** - Sends email via SMTP2GO (auto-creates HubSpot ticket), includes AI insights and full transcript

### Session Tracking
- Tracks: questions_asked, kb_articles_viewed, tickets_created
- Activity timeline with timestamps
- Customer info capture (name, email) during ticket creation

### Design System
- **Responsive:** Mobile-first with breakpoints: xs(475px), sm(640px), md(768px), lg(1024px), xl(1280px), 2xl(1536px)
- **Theme:** Dark/light mode with HSL color system
- **Animations:** fade-in, slide-in-bottom, scale-in, shimmer, pulse-subtle
- **Shadows:** Layered shadow system (subtle, card, elevated, high)
- **Glass Morphism:** backdrop-blur with transparency

## Important Conventions

### Frontend Styling
- **NEVER** use emojis unless explicitly requested
- Use Tailwind utility classes with responsive breakpoints
- Apply shadcn/ui components for consistency
- All animations use cubic-bezier easing
- Glass morphism for headers and overlays

### Backend Patterns
- All tools must call `state_manager.log_*()` methods
- HubSpot integration has demo mode (no API key needed)
- Tickets are logged to `logs/created_tickets.json`
- Session data stored in memory (not persistent)

### Git Workflow
- `.gitignore` excludes: `.claude/`, `*.csv`, `hubspot_tickets_*.csv`
- CSV data files should NEVER be committed
- Always run type checks before committing: `pnpm typecheck`

## Environment Variables

### Backend (.env in backend/)
```env
OPENAI_API_KEY=sk-proj-...           # Required
SMTP2GO_API_KEY=api-...              # Required for email escalation
ZING_SUPPORT_MODEL=gpt-5.1           # Optional (default: gpt-5.1)
ALLOWED_ORIGINS=http://localhost:5171  # CORS origins (comma-separated)
```

### Frontend (.env in frontend/)
```env
VITE_SUPPORT_API_BASE=http://localhost:8001/support
VITE_SUPPORT_CHATKIT_API_URL=http://localhost:8001/support/chatkit
VITE_SUPPORT_CUSTOMER_URL=http://localhost:8001/support/customer
VITE_SUPPORT_CHATKIT_API_DOMAIN_KEY=any-value-for-local  # Required (any value for local dev)
```

### Changing Theme Colors
Update HSL values in `frontend/src/index.css` under `:root` and `.dark` sections.

## Troubleshooting

### "ChatKit has excessive padding"
Override with CSS in `index.css` - see lines 274-325 for ChatKit-specific overrides.

### "Session stats not updating"
Verify tools in `zing_support_agent.py` are calling `state_manager.log_*()` methods.

### "HubSpot tickets not creating"
Check if `HUBSPOT_API_KEY` is set. Demo mode works without it but logs to `logs/created_tickets.json`.

### "Frontend not responsive"
Check Tailwind breakpoint classes. Pattern: `class="xs:col-span-2 sm:col-span-3 lg:col-span-4"`

## Production Deployment

### Backend
1. Set environment variables on hosting platform
2. Deploy FastAPI app (Railway, Render, AWS Lambda, etc.)
3. Configure CORS for frontend domain in `main.py`

### Frontend
1. Register domain on [OpenAI Domain Allowlist](https://platform.openai.com/settings/organization/security/domain-allowlist)
2. Set `VITE_SUPPORT_CHATKIT_API_DOMAIN_KEY` to generated key
3. Update `vite.config.ts` with production domain
4. Build and deploy (Vercel, Netlify, Cloudflare Pages, etc.)
